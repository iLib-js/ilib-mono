loadFile(pathName, options) → {Promise|string|undefined}

Description:
  • Load an individual file specified by the path name, and return its content. If the file does not exist or could not be loaded, this method will return undefined.

    The options object may contain any of these properties:

    • sync {boolean} - when true, load the file synchronously, else load it asynchronously. Loaders that do not support synchronous loading will ignore this option.
    For files that end with a ".js" or ".mjs" extension, this method should treat the file as a Javascript module and load it accordingly. All other file types will be loaded as UTF-8 text.

loadFiles(paths, options) → {Promise|Array.<string>|undefined}

Description:
  • Load a number of files specified by an array of file names, and return an array of content. The array of content is in the same order as the file names such that the n'th element of the return array is the content of the file with the n'th file name in the paths parameter. If any particular file does not exist or could not be loaded, that entry in the return array will be undefined.

    The options object may contain any of these properties:

    • sync {boolean} - when true, load the files synchronously, else load them asynchronously. Loaders that do not support synchronous loading will ignore this option.
    The loadFiles method depends on the subclass to implement the abstract method loadFile to load individual files.
