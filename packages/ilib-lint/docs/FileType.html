FileType

FileType

Represent a type of file in an ilib-lint project. Each file is classified into a particular file type. If none of the file type definitions match, then the file will be classified as being in the default "unknown" file type. Files in the unknown file type are usually not processed.

Constructor

new FileType(options)

Description:
  • Contructor a new instance of a file type. The array of parsers will be used to attempt to parse each source file. If a parser throws an exception/error while parsing, the linter will note that an error occurred and move on to the next parser to see if that one will work. If ALL parsers fail for a particular file, then this tool will print an error message to the output about it.

Properties
Name Type Attributes Description
name String the name or glob spec for this file type
project Project the Project that this file type is a part of
locales Array.<String> <optional>
 list of locales to use with this file type
template String <optional>
 the path name template for this file type which shows how to extract the locale from the path name if the path includes it. Many file types do not include the locale in the path, and in those cases, the template can be left out. If a serializer is specified, then the template is also used by the serializer to determine how to name the output files.
parsers Array.<String> <optional>
 an array of names of parsers to apply to this file type. The first parser is the most important one, as its type will be used to determine the type of intermediate representation, the type of the rules, and the type of the transformers and serializer. If no parsers are specified, then the parser manager will be asked to find all parsers that can parse files of this type.
ruleset Array.<String> <optional>
 a list of rule set names to use with this file type. Only rules in these rule sets that operate on the same type of intermediate representation as the parsers will be applied to the file.
transformers Array.<String> <optional>
 an array of transformer names to apply to files of this type after the rules have been applied. Every transformer must operate on the same type of intermediate representation as the parser.
serializer String | Object <optional>
 the name of the serializer to use if the file has been modified by a transformer or a fixer. The serializer must operate on the same type of intermediate representation as the parser.
Throws:
if a transformer or serializer is specified that does not operate on the same type of intermediate representation as the parser, or if a parser, transformer, or serializer cannot be found.
