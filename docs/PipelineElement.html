PipelineElement

superclass for pipeline elements

A pipeline element is the superclass for all classes that can be used in a pipeline. A pipeline element can be used along with a number of other pipeline elements of the same type to process a particular type of file. Each pipeline element should define a type, a name, and a description.

(abstract, readonly) type :string

Description:

  • Type of file that this pipeline element processes. The type should be a unique name that matches with the type of other pipeline elements that process this same type of file.

    There are three types that are reserved, however:

    • resource - the pipeline element returns an array of Resource instances as defined in https://github.com/ilib-js/ilib-tools-common.
    • line - the pipeline element produces a set of lines as an array of strings
    • string - the pipeline element doesn't parse. Instead, it just treats the the file as one long string.

    Concrete subclasses must define this property.
